TestScribe Documentation

TestScribe is a developer-focused Chrome extension designed to speed up QA and developer workflows by automatically generating high-quality Gherkin (Cucumber) and Manual test cases from Jira ticket details. It integrates directly with local models (via Ollama) or cloud providers (Gemini, Groq, custom OpenAI hosts) and uploads results directly to Jira or Xray Cloud.

Multi-LLM Native

Run locally using Ollama (free & secure) or leverage cloud intelligence from Gemini, Groq, or any OpenAI-compatible server.

Step Consistency

Enforce clean Gherkin standards by matching generated steps with your team's existing step dictionary using fuzzy Jaccard similarity.

Installation

Since TestScribe is currently distributed as an unpacked developer package, follow these quick steps to load it into Chrome:

  1. Download or clone the TestScribe repository files.
  2. Open Chrome and navigate to chrome://extensions/.
  3. Enable Developer mode using the toggle switch in the top-right corner.
  4. Click Load unpacked in the top-left menu.
  5. Select the root folder containing the extension's files.
Success
The TestScribe pen icon will appear in your extensions list. Pin it to your address bar for fast access!

Setup: Ollama (Local AI)

Ollama runs entirely on your local CPU/GPU, keeping your Jira data private and secure.

1. Download Ollama

Install the desktop application from ollama.com.

2. Pull Coder Model

We recommend qwen2.5-coder for its precise structured output capability:

ollama pull qwen2.5-coder

3. Configure CORS Access

Chrome extensions run under custom web-origins. You must configure Ollama to accept connection headers from extension origins:

macOS: 
launchctl setenv OLLAMA_ORIGINS "*"

After running, completely quit the Ollama menu bar app and restart it.

Windows:

Open environment variables configurations and create a new User variable:

  • Variable Name: OLLAMA_ORIGINS
  • Variable Value: *

Restart the Ollama service to apply the change.

4. Save Settings

In TestScribe Settings, choose Ollama (Local), hit Refresh to view your pulled models, select the model, and click Save.

Setup: Gemini (Cloud AI)

Quick cloud configuration leveraging Google Gemini's models.

  1. Go to Google AI Studio and generate a free API key.
  2. Open TestScribe Settings → Select **Gemini (Cloud)**.
  3. Paste your API key in the password field and hit **Save Agent Settings**.

Setup: Groq (Cloud AI)

Enables ultra-fast cloud model generations using Llama 3, DeepSeek, or Mixtral models.

  1. Generate an API key on the Groq Console.
  2. In Settings, select **Groq (Cloud)** and input your API key.
  3. Click **Refresh** to load models (e.g. llama-3.3-70b-specdec) and choose your preferred backend.
  4. Save settings.

Setup: Custom OpenAI Endpoint

Connect TestScribe to local servers (LM Studio, LocalAI) or custom API gateways.

  1. Select **Custom OpenAI** in settings.
  2. Set the **Custom Endpoint Host URL** (e.g., http://localhost:1234/v1 for LM Studio).
  3. Add an API key if required by your provider.
  4. Click **Refresh** to populate the models list from the server's /models endpoint and select one.

Usage Guide

TestScribe integrates inline tabs for both Gherkin (BDD) and manual test case generation:

  1. Open a Jira ticket page (e.g., https://*.atlassian.net/browse/TS-101).
  2. Click the TestScribe extension icon. The extension automatically loads ticket fields.
  3. Choose a generator tab (Gherkin Prompt or Manual Prompt).
  4. Click **Generate Test Cases** to fetch responses.

For each generated card, you can:

  • **Copy**: Copy Gherkin code to clipboard.
  • **Edit**: Type inline modifications inside step textareas.
  • **Create JIRA / Xray Test**: Upload and link it to the current requirement ticket.

Context Enrichment

Context enrichment lets the AI read related ticket hierarchies to generate highly relevant steps matching your domain vocabulary.

  • Epic/Story Context: Auto-scans for parent issues and retrieves descriptions to append to prompts.
  • Interactive Context Toggle Panel: Discovered parent relations appear in a checklist between the title and prompt editor, enabling manual attachment control.
  • Enrichment Settings: Configure settings to "Include parent ticket by default" or toggle the popup panel hide option.

Review & Merge (Step Consistency)

Enforce step vocabulary consistency across test repositories before uploading new scenarios:

  • Xray Step Library Scanning: TestScribe connects to the Xray Cloud GraphQL API using your Client ID and Client Secret. When you trigger a sync on the Integration tab, it scans all issues of type Test within your selected Jira projects, filters for Cucumber (Gherkin) tests, extracts their scenario definition steps, and stores them in the extension's local storage database. (Note: Manual tests are bypassed to keep focus strictly on reusable BDD steps).
  • Fuzzy Matcher: Uses Jaccard similarity metrics to check generated steps against synced library steps. High matches show up in green, warning of slight variations that can be normalized.
  • Step Autocomplete: When editing steps, start typing to see autocomplete recommendations from the library step dictionary.
  • Prompt Context Injection: The synced step library is also used during generation. TestScribe finds similar existing Gherkin steps based on the ticket's title and description and appends them directly to the LLM system prompt, guiding the AI to reuse existing vocabulary naturally.
  • Save & Merge Target: Compares step sets against available test tickets and auto-selects the best-matching ticket to append new scenarios.

Troubleshooting

Error Message Likely Cause Resolution
403 Forbidden Ollama CORS blocked Set OLLAMA_ORIGINS="*" environment variable and restart Ollama.
Ollama unreachable Ollama not running Verify the Ollama process is active and check the Host URL in settings.
No JIRA data detected Invalid page URL Make sure you are on a Jira ticket browse view (URL path contains /browse/).
